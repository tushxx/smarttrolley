2. Detailed Data Flow (Step-by-Step)
Phase 1: User Login & Session Start
Shopper opens store mobile app → generates unique session QR code

QR displayed on phone → Shopper holds it in front of cart's Pi camera

Pi camera captures QR → qr_login.py decodes session ID

Session validated → Cart LED turns green, touchscreen shows "Welcome"

Session data sent to cloud → AWS IoT Core logs cart activation

Phase 2: Item Addition Flow
Shopper picks item → presents barcode to fixed-mount scanner

Barcode scanner reads SKU → barcode_service.py receives USB data

Item dropped in basket → weight increases on load cell platform

Dual cameras capture event → vision_service.py runs YOLO-tiny inference

Validation logic executes:

python
expected_weight = product_db[barcode]['weight']
weight_match = abs(actual_delta - expected_weight) <= 5g
vision_match = (yolo_label == product_db[barcode]['name'])

if weight_match and vision_match:
    status = "SUCCESS"
else:
    status = "FRAUD_ALERT"
Event published to AWS → MQTT message to cart/{cart_id}/item_add

AWS Lambda processes → updates DynamoDB cart session

UI updates → item appears on screen with running total

Phase 3: Fraud Detection & Alerts
Anomaly detected (weight without scan, wrong weight, etc.)

Cart triggers local alert → red LED, buzzer sound

Fraud event published → cart/{cart_id}/alert_fraud topic

Lambda fraud handler → increments fraud counter in DynamoDB

SNS notification sent → store manager receives mobile alert

After 3 fraud events → checkout button locked until staff override

Phase 4: Item Removal Flow
Shopper taps "Remove" on UI → removal mode activated

Item rescanned → barcode confirms which item to remove

Weight decreases → load cell validates removal

Removal event published → cart/{cart_id}/item_remove

Lambda updates → DynamoDB cart total recalculated

UI refreshes → item removed from display

Phase 5: Checkout & Payment
Shopper taps "Checkout" → checkout request sent to cloud

Lambda calculates final bill → applies taxes, discounts

Payment QR generated → UPI/Razorpay payment link created

QR displayed on screen → shopper scans with payment app

Payment confirmed → webhook triggers receipt generation

Receipt stored in S3 → PDF generated and emailed to customer

Session closed → cart resets for next user

3. Component-Level Architecture
Edge Device (Raspberry Pi 4)
text
Hardware Layer:
├── Pi 4B (4GB) - Main controller
├── 2x Pi Camera v3 (8MP) - YOLO vision
├── Dcode DC5132 - Barcode scanner
├── YZC-131 + HX711 - Load cell system
├── 7" HDMI Touch Display - User interface
├── RGB LED ring + Buzzer - Feedback
└── 10,000mAh Power bank - 8h runtime

Software Services:
├── qr_login.py - Session management
├── barcode_service.py - USB scanner interface
├── vision_service.py - YOLO inference
├── weight_guard.py - Load cell monitoring
├── fraud_guard.py - Anomaly detection
├── ui_app.py - Kivy touch interface
└── mqtt_client.py - AWS IoT connectivity
AWS Cloud Stack
text
IoT & Messaging:
├── AWS IoT Core - Device registry, MQTT broker
├── AWS IoT Rules - Message routing
└── AWS IoT Jobs - OTA updates

Compute:
├── AWS Lambda Functions:
│   ├── handleAddItem() - Process item additions
│   ├── handleRemoveItem() - Process removals
│   ├── handleCheckout() - Generate payment
│   └── fraudNotifier() - Alert management

Storage:
├── Amazon DynamoDB:
│   ├── Products table - SKU, price, weight
│   ├── CartSessions table - Active carts
│   └── Receipts table - Transaction history
└── Amazon S3:
    ├── Receipt PDFs
    ├── Camera frames (audit)
    └── System logs

Integration:
├── Amazon SNS - Push notifications
├── Amazon API Gateway - REST endpoints
├── Amazon Cognito - User authentication
└── Amazon QuickSight - Analytics dashboard
4. Security & Communication Flow
Device Security
TLS 1.2 encryption for all MQTT communication

X.509 certificates for device authentication

IAM policies restrict topic access per cart

Secure boot prevents firmware tampering

Data Privacy
On-device processing for camera frames (no raw images sent to cloud)

Encrypted storage for session tokens

GDPR compliance with automatic data purging after 30 days
